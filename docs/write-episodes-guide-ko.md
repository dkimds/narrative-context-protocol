# 바이브 코딩으로 숏폼 드라마 쓰기 — 시작 가이드

> 코딩 경험 없이도 AI와 대화하면서 드라마 시리즈를 만들 수 있습니다.

---

## 이게 뭔가요?

`/write-episodes`는 **클로드(AI)와 대화해서 숏폼 드라마 시리즈를 회차별로 함께 쓰는 도구**입니다.

당신이 할 일: 드라마 아이디어와 이야기 방향을 말로 설명한다.
클로드가 할 일: 회차별 내용을 제안하고, 구조화된 파일로 자동 저장한다.

최종 결과물:
- `productions/ep01.json`, `ep02.json` … — 각 에피소드 파일 (드라마 구조 저장 형식)
- `productions/series-constants.json` — 인물과 설정 상수
- `productions/[시리즈명].pdf` — 기획서 PDF

> **JSON이 뭔가요?** 텍스트로 된 구조화된 저장 형식입니다. 직접 열어보거나 편집할 필요는 없습니다. 클로드가 모두 처리합니다.

---

## 준비물

### 1. Visual Studio Code 설치

[code.visualstudio.com](https://code.visualstudio.com) 에서 무료 다운로드.

### 2. Claude Code 확장 설치

VS Code 왼쪽 Extensions 탭(네모 아이콘) → `Claude Code` 검색 → Install.

### 3. Node.js 설치

[nodejs.org](https://nodejs.org) 에서 LTS 버전 다운로드 후 설치.

> Node.js는 `npm run ep`, `npm run pdf` 같은 명령을 실행하는 데 필요합니다.

### 4. 이 리포지토리 받기

VS Code 터미널 열기 (메뉴 → Terminal → New Terminal) 후 아래 입력:

```
git clone https://github.com/[리포 주소].git
cd narrative-context-protocol
npm install
```

> `npm install`은 처음 한 번만 하면 됩니다. 드라마 도구가 사용하는 부품들을 설치합니다.

---

## 시작하기

VS Code에서 이 폴더를 열고, Claude Code 채팅창에 아래를 입력합니다:

```
/write-episodes
```

그러면 클로드가 한국어로 질문을 시작합니다.

---

## 대화 흐름

### 1단계: 시리즈 설정 (처음 한 번)

클로드가 7가지를 물어봅니다. 아래는 예시 답변입니다:

| 질문 | 예시 답변 |
|------|-----------|
| 시리즈 제목 | 공항 분실물 센터 |
| 전체 회차 목표 | 총 40회, 지금은 1~5회 |
| 장르 | 로맨스, 코미디 |
| 주인공(MC) | 차윤서, 29세 분실물 센터 직원 |
| 영향력 캐릭터(IC) | 류정호, 35세 상습 분실 신고자 |
| 결말 방향 | 해피엔딩 |
| 에피소드 길이 | 90초 |

답변하면 클로드가 **시리즈 바이블**을 정리해서 보여줍니다. 확인하면:

- `productions/series-constants.json` 자동 저장 (인물별 고유 번호 생성)

> **인물을 더 추가하고 싶으면?** "조연 한 명 더 추가하자. [이름], [역할]" 이라고 말하면 됩니다.

---

### 2단계: 에피소드 작성 (회차마다 반복)

**A. 클로드가 한글로 회차 내용 제안**

```
## 1회 — 오늘도 오셨네요

기 (설정, 15초): 바쁜 오후, 윤서가 분실물 창구를 정리하던 중
승 (갈등, 35초): 정호가 황당한 물건을 찾아달라며 신고서를 내민다
전 (반전, 25초): 알고 보니 그 물건에 진짜 사연이 있었다
결 (여운, 15초): 물건을 찾아줬더니 다음날 또 나타난 정호

이번 회차 핵심 감정: 황당함
다음 회차 연결고리: 정호가 매번 찾는 물건, 사실 핑계인가?
```

**B. 피드백 주기**

내용이 마음에 들면 "좋아, 진행해" / 수정이 필요하면 "승 부분을 더 코믹하게 바꿔줘" 식으로 말합니다.

**C. 자동 저장 및 검증**

클로드가:
1. `productions/ep01.json` 파일 자동 저장
2. 구조 오류가 있는지 자동 검사
3. `PASS` 또는 오류 내용 보고

**D. 다음 회차**

"다음 회차 진행할까요?" 에 "응" 하면 2회로 넘어갑니다.

---

## 저장되는 파일

```
productions/
├── series-constants.json   ← 인물 정보와 고유 번호 (시리즈 전체 공통)
├── ep01.json               ← 1회 에피소드
├── ep02.json               ← 2회 에피소드
├── ...
└── 공항-분실물-센터.pdf     ← 전체 기획서 PDF
```

> 이 폴더가 없으면 자동으로 생성됩니다.

---

## 에피소드 검증 (선택)

저장된 에피소드가 형식에 맞는지 직접 확인하고 싶을 때:

```
npm run ep -- 1        ← 1회 검증
npm run ep -- all      ← 전체 검증
```

`PASS productions/ep01.json` 메시지가 나오면 정상입니다.

---

## PDF 기획서 만들기

```
npm run pdf
```

제목과 파일명은 ep01.json에서 자동으로 가져옵니다.
직접 지정하고 싶으면:

```
npm run pdf -- "내 드라마 제목"
npm run pdf -- "내 드라마 제목" "파일명"
```

`productions/[파일명].pdf` 파일이 생성됩니다. 더블클릭으로 바로 열립니다.

## 자주 묻는 질문

**Q. 코딩을 전혀 몰라도 되나요?**
네. 대화만 하면 됩니다. `npm install`, `npm run pdf` 같은 명령은 터미널에 그대로 붙여넣으면 됩니다.

**Q. 클로드가 제안한 내용이 마음에 안 들면요?**
"전 부분을 반전 없이 감정적으로만 끝내줘" 처럼 구체적으로 말하면 수정해줍니다. 몇 번을 수정해도 괜찮습니다.

**Q. 인물을 중간에 추가할 수 있나요?**
가능합니다. 에피소드 작성 중 "새 인물 추가하자. [이름], [설명]" 이라고 말하면 클로드가 `series-constants.json`에도 자동으로 추가합니다.

**Q. 오류가 났을 때 어떻게 하나요?**
`npm run ep` 결과에 오류 내용이 나오면 그대로 클로드 채팅창에 붙여넣고 "이 오류 고쳐줘"라고 하면 됩니다.

**Q. 새 대화를 시작해도 이전 에피소드를 이어서 쓸 수 있나요?**
네. `/write-episodes` 를 다시 호출하면 클로드가 `productions/series-constants.json`과 기존 에피소드 파일을 읽어서 이어서 진행합니다.

**Q. 40회를 한 번에 다 써야 하나요?**
아니요. 1~5회 쓰고 멈춰도 됩니다. 나중에 다시 이어서 쓸 수 있습니다.
